# Day 6 勉強用資料

## 今日のテーマ
**APIとAI連携の全体像をつかみ、入力 → 処理 → 出力 の流れを理解する**

---

## 1. 今日の結論
Day6で大事なのは、
**AIや外部サービスは、情報を渡すと処理して返してくれる仕組みで動いている**
と理解することです。

今日は特に、この3つをつかめればOKです。
- API はサービス同士をつなぐ窓口
- 処理の流れは「入力 → 処理 → 出力」
- AI活用や自動化は、この流れの組み合わせでできている

---

## 2. API とは何か
APIは、
**別のサービスにお願いを送って、結果を返してもらう仕組み**
です。

人にたとえると、
- 自分がお願いを出す
- 相手がそれを受け取る
- 相手が処理して返事を返す

という流れです。

### たとえば
- ChatGPTに文章を作ってもらう
- Googleスプレッドシートにデータを書き込む
- 天気サービスから情報を取ってくる

こういうものは、裏でAPIが動いていることが多いです。

---

## 3. いちばん大事な流れ
APIやAI連携では、まずこの流れを覚えるのが大事です。

### 入力 → 処理 → 出力

#### 入力
何を渡すか

#### 処理
相手が何をするか

#### 出力
何が返ってくるか

この3つで考えると、かなり整理しやすくなります。

---

## 4. ChatGPTのイメージ
たとえば、ChatGPTに
「Instagram投稿案を3本作って」
と頼む場合はこう考えます。

### 入力
- テーマ
- ターゲット
- トーン
- 本数

### 処理
- AIが内容を理解する
- 条件に合わせて文章を作る

### 出力
- 投稿案3本

つまり、
**AIは魔法ではなく、渡した情報をもとに処理して返している**
と考えると分かりやすいです。

---

## 5. Googleスプレッドシートのイメージ
たとえば、広告データを自動で記録する場合です。

### 入力
- 日付
- 媒体名
- クリック数
- CV数

### 処理
- API経由でシートに書き込む

### 出力
- シートにデータが追加される

これは、
**AIではなくても API の考え方は同じ**
という例です。

---

## 6. AI連携の実務イメージ

### SNS運用
- 入力: テーマ、ターゲット、媒体
- 処理: AIが投稿案を作る
- 出力: 投稿文の候補

### 広告運用
- 入力: 商品情報、訴求軸、ターゲット
- 処理: AIが広告文を考える
- 出力: 広告文案

### 営業
- 入力: 業界、悩み、提案したいサービス
- 処理: AIが提案文のたたきを作る
- 出力: 提案メッセージ案

---

## 7. ノーコードとコードの違い
ここもよく大事になります。

### ノーコード
- 画面上で設定してつなぐ
- 直感的で始めやすい
- 小さな自動化ならかなり便利

### コード
- 自由度が高い
- 細かい条件分岐や複雑な処理に強い
- カスタマイズしやすい

でも本質は同じで、どちらも
**入力 → 処理 → 出力**
を組んでいます。

---

## 8. APIで考えるコツ
APIやAI連携を考えるときは、
いきなり難しく考えなくて大丈夫です。

まずは次の3つに分けると整理しやすいです。
- 何を渡すのか
- 相手に何をさせたいのか
- 最後に何がほしいのか

これだけでも、かなり実務に使える考え方になります。

---
