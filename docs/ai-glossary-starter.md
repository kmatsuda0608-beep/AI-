# 最初に覚えるAI・プログラミング用語集

## AI
人のように文章生成・要約・分類・提案などを行う仕組み全般。

## LLM
大量の文章を学習して、自然な文章を生成する大規模言語モデル。ChatGPT系の土台。

## プロンプト
AIに出す指示文。AIの出力品質を左右する重要要素。

## API
サービスとサービスをつなぐための窓口。AIを他ツールと連携するときによく使う。

## トークン
AIが文章を処理するときの細かい単位。ざっくり「文字数や単語数に近い計算単位」と考えてよい。

## 自動化
人が毎回手作業でやっている流れを、ツールやコードで省力化すること。

## Python
AI・データ処理・自動化で使われやすいプログラミング言語。読みやすく、初心者向き。

## JavaScript
Webサイトやブラウザ側でよく使う言語。将来的にWeb制作や画面操作にも強い。

## 変数
データを入れておく箱。名前をつけて、後から使い回せる。

## 条件分岐
「もし〜なら」を表現する考え方。状況によって処理を変える。

## 繰り返し処理
同じ作業をまとめて何回も実行する考え方。

## データ構造
情報をどう整理して持つかという考え方。AI活用でもかなり重要。

## CSV
表形式データを簡単に扱えるファイル形式。広告運用やデータ整理でよく登場する。

## RAG
外部情報を検索して、その内容をもとにAIに答えさせる仕組み。社内ナレッジ活用などで使われる。

## fine-tuning
AIモデルを追加学習して特定用途向けに調整すること。最初の段階では必須ではない。

## ノーコード
コードを書かずにツール同士をつなぐ方法。

## ローコード
少しだけコードを書きながら仕組みを作る方法。

## 学習のポイント
- 最初は定義を完璧に暗記しなくてよい
- 「これは何のための言葉か」を掴むのが先
- 実務の場面とセットで覚えると定着しやすい
